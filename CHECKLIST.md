## Перевірка перед деплоєм

### 1. Локальна перевірка
```bash
# Встановіть залежності
npm install

# Перевірте збірку
npm run build

# Перевірте preview
npm run preview
```

### 2. Git налаштування
- [ ] Ініціалізовано Git репозиторій
- [ ] Всі файли додані до Git
- [ ] Створено commit
- [ ] Репозиторій створено на GitHub
- [ ] Локальний репозиторій підключено до GitHub

### 3. Vercel налаштування
- [ ] Зареєстровано/увійдено на Vercel
- [ ] Підключено GitHub акаунт
- [ ] Імпортовано репозиторій
- [ ] Перевірено автоматичні налаштування:
  - Framework: Vite
  - Build Command: `npm run build`
  - Output Directory: `dist`
- [ ] Додано змінні оточення (якщо потрібно):
  - `GEMINI_API_KEY` (опціонально)

### 4. Після деплою
- [ ] Сайт відкривається без помилок
- [ ] Всі сторінки працюють
- [ ] Навігація працює (SPA routing)
- [ ] Статичні ресурси завантажуються
- [ ] API ключі працюють (якщо використовуються)

## Можливі проблеми та рішення

### Проблема: Помилка збірки на Vercel
**Рішення:**
- Перевірте логи в Vercel Dashboard
- Переконайтесь, що Node.js версія 18+
- Перевірте, що всі залежності в `package.json`

### Проблема: Сторінка не знайдена (404) при навігації
**Рішення:**
- Перевірте, що `vercel.json` містить rewrites
- Переконайтесь, що всі роути ведуть до `/index.html`

### Проблема: Змінні оточення не працюють
**Рішення:**
- Перевірте назви змінних (чутливі до регістру)
- Переконайтесь, що змінні додані для правильного середовища (Production)
- Перезапустіть деплой після додавання змінних

## Наступні кроки

1. **Після успішного деплою:**
   - Налаштуйте кастомний домен (опціонально)
   - Налаштуйте автоматичні деплої для інших гілок
   - Додайте моніторинг (опціонально)

2. **Оптимізація:**
   - Додайте analytics (Google Analytics, Plausible, тощо)
   - Налаштуйте CDN для статичних ресурсів
   - Оптимізуйте зображення

3. **Безпека:**
   - Перевірте, що API ключі не потрапляють у код
   - Використовуйте змінні оточення для чутливих даних
   - Налаштуйте CORS якщо потрібно
